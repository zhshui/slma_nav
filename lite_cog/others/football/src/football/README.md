目前读取图像的方法是usb-cam驱动，图像的大小是640*480，real_sense驱动是424×240
detect_ball.cpp：
通过把rgb->hsv，过滤颜色，仅留下红色区域，然后检测轮廓，留下最大轮廓，最大轮廓面积大于150,输出为目标物体->小球。
为了在小球不出现时，过滤掉视野的其他红色物体，采用了圆度的判断afa，afa越小越可能是圆。远处的物体判断较差，一般阈值为0.7,0.6...
调整的参数：hsv图像辨别红色，容易与黑色误检，所以调整v信道，具体视环境情况而定，v>46至少。h和s信道（s>43至少）也可调整。范围越小越好，但是也会造成有些情况检测不出红色球。
control_ball.cpp：
控制球的代码，采用最简单的比例控制P：小球中心点与图像中心距离越远，角速度越大，始终保持x在像素坐标系中间，左侧左转，右侧右转;小球半径r决定移动的线速度，r>threshold_r，远离，r<threshold_r，靠近，相差越大，移动越快;比例大小×最大速度即为最终速度;最大线速度maxv,最大角速度maxr
WIDTH，HEIGHT是图像的大小，随图像大小改变，同时maxr,maxv也要调整，半径的阈值threshold_r也要调整;发送的移动指令数据格式Twist,发送到/cmd_vel topic;
control_ball_combine.cpp：
两种相机合并控制，远距离用usb-cam，近距离用real_sense，走到面前后停止运动，flag=0
